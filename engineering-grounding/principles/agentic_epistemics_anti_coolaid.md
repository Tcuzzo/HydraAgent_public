# Agentic-Builder Epistemics — Anti-Cool-Aid Grounding

Fluent output is not evidence. Before any claim ships, it must trace to source truth, survive a falsification attempt, and carry honest confidence. These guardrails are load-bearing at build time.

- **Resolve every symbol against the installed artifact before you write it.** — Read the module, run `--help`, or grep the pinned tree; if you can't point to where an import/signature/flag/env/key is defined *right now*, it's a hallucination, not a guess to ship (USENIX Sec 2025 / arXiv:2406.10279: 5.2–21.7% of LLM-generated packages don't exist). Guards against hallucinated/fabricated APIs and slopsquatting supply-chain hijacks.
- **Fetch and quote the primary source before you rely on it.** — Any doc clause, spec, error code, changelog line, or "the API says" must be retrieved and quoted verbatim; if it can't be pulled, mark it UNVERIFIED and get it first — never reconstruct from memory (Walters & Wilder, *Sci. Reports* 2023: 55% of GPT-3.5, 18% of GPT-4 citations fabricated). Guards against confident fabricated citations/quotes justifying a design.
- **Reproduce from clean state twice, with evidence, before reporting green.** — Fresh checkout/env + pinned seed, command and output attached; a result you can't re-run from scratch is folklore, not proof (Collberg & Proebsting, CACM 2016: only ~32% of code-backed papers even built in 30 min; ACM Artifact Badging). Guards against one-shot / "works on my machine" green.
- **Trace any SE maxim to its primary study before you build on it.** — "10x devs," "100x cost-to-fix-later," the cone of uncertainty, "framework X is always faster" — confirm the original actually measured what the slogan claims (Bossavit, *Leprechauns of Software Engineering* 2015: these are citation-telephone folklore). Guards against cargo-cult SE folklore baked into architecture.
- **Mark every claim verified vs. inferred, and treat low confidence as a trigger to go verify — never trust a model's self-reported confidence as "calibrated" without an external check.** Say "I know this" vs "I'm inferring"; when unsure, verify against source truth rather than smoothing into one confident register. RLHF-aligned models are documented to be systematically OVERCONFIDENT and to lose calibration after preference tuning (OpenAI GPT-4 System Card; Kadavath et al. arXiv:2207.05221 show P(IK) is base-model- and prompt-fragile), so a confidence number is trustworthy only after an independent verification step — never self-graded. Guards against treating fluent self-assessed confidence as calibrated truth (false comfort), and against a harness grading its own certainty.
- **Enumerate ≥2 hypotheses and run the cheapest excluding test before editing.** — Strong inference: kill a rival cause first, then edit; a green run *after* your change is correlation, not proof it was the cause (Platt, *Science* 1964). Guards against single-hypothesis / post-hoc "my fix worked" debugging.
- **Try to falsify your own change: run the test you expect to FAIL.** — Exercise adversarial, boundary, and negative/error cases, not just the happy path you built for; knowledge advances by refutation, and confirmation-seeking is the default failure mode (Popper 1963; Wason 1960, 2-4-6 task). Guards against confirmation-seeking verification treating happy-path green as proof.
- **Demand a measured order-of-magnitude win on your real workload before rebuilding around a "game-changer."** — No single tool/framework/model has delivered 10x in a decade; new tooling only shaves *accidental* complexity — the essential complexity gets re-paid regardless (Brooks, *No Silver Bullet* 1986/87). Guards against silver-bullet / fad-driven rebuilds on marketing claims.
- **Re-evaluate on fresh, held-out, task-representative cases — never trust a headline benchmark.** — Eval sets leak into training (contamination inflates non-transferable scores) and any targeted metric decays as a proxy (arXiv:2406.04244; Strathern 1997 on Goodhart: "when a measure becomes a target, it ceases to be a good measure"). Guards against benchmark-overfit / contamination-inflated proof of capability.
